# TalentPlatform: анализ ТЗ и план реализации

## 1) Свод требований из документов

### Основные функции
- Роли: студент, преподаватель, методист, куратор (и администратор).
- Регистрация/вход/выход, разграничение доступа.
- CRUD проектов, этапов, комментариев.
- Статусы проекта: планируется, в работе, на проверке, завершен, отменен.
- Поиск, фильтрация, сортировка.
- Пагинация на backend.
- Портфолио и рейтинг активных участников.
- Аналитика и диаграммы.
- Публикация завершенных проектов, лайки, комментарии с модерацией.

### Нефункциональные требования
- SPA + REST API.
- PostgreSQL (3НФ).
- Токены в HttpOnly cookies, без localStorage для секретов.
- CORS только для доверенных доменов.
- Логирование в консоль backend.
- Адаптивный UI.

### Дополнительные требования (на “отлично”)
- Экспорт/импорт (xlsx/pdf/docx), отчеты, диаграммы.
- Смена/восстановление пароля через email.
- Деплой в Docker (будет позже).
- Желательно NoSQL для логов/чатов.

## 2) Что уже реализовано в этом шаге

### Backend (Django + DRF)
- Создан каркас проекта: `backend/manage.py`, `backend/config/*`.
- Настроены:
  - PostgreSQL,
  - DRF + фильтры + пагинация,
  - CORS/CSRF trusted origins,
  - OpenAPI/Swagger (`/api/docs/`),
  - консольное логирование.
- Реализована auth-модель:
  - кастомный пользователь с ролями (`apps.users.models.User`),
  - endpoints: `login/logout/refresh/me`,
  - JWT в HttpOnly cookies.
- Реализованы проекты:
  - модели: `Project`, `ProjectStage`, `ProjectComment`, `ProjectLike`,
  - фильтрация, поиск, сортировка, пагинация,
  - модерация комментариев куратором,
  - публикация проекта куратором.

### Frontend (React SPA)
- Добавлены:
  - роутинг (`react-router-dom`),
  - auth context и защищенные маршруты,
  - страницы: Login, Dashboard, Projects, Project detail,
  - базовый API client (`axios`, `withCredentials: true`),
  - новый адаптивный UI-каркас.

## 3) Архитектура по этапам (рекомендуемо)

### Этап A (текущий MVP)
- Auth + роли.
- Проекты + этапы + модерация комментариев.
- Поиск/фильтры/сортировка/пагинация.

### Этап B
- Массовый импорт пользователей из Excel для куратора.
- Уведомления по событиям (создание проекта, отправка этапа, модерация комментариев).
- Профили и портфолио пользователей.

### Этап C
- Аналитика для методиста (диаграммы, срезы по периодам).
- Экспорт PDF/XLSX/DOCX.
- Восстановление пароля по email.

### Этап D
- NoSQL для чатов/логов событий (например MongoDB/Redis).
- Docker Compose (frontend/backend/postgres/redis).

## 4) Важные замечания по вашему ТЗ

- “NoSQL для логов и чатов” и “PostgreSQL в 3НФ” не конфликтуют: 
  - транзакционные данные хранить в Postgres,
  - чат/журнал событий вынести в отдельное NoSQL-хранилище.
- Генерация логина/пароля при загрузке Excel делается сервисом импорта + отправкой данных по email.
- Для защиты маршрутов нужен двойной контроль:
  - frontend guarded routes,
  - backend permissions (уже заложено).
